INTRODUCTION

The NYU research team proposes utilizing Wi-Fi and Bluetooth technology to estimate origin-destination demands and wait times of pedestrians. If the system is equipped with Wi-Fi and Bluetooth receivers, it is possible to capture O-D information for some percentage of the pedestrians with visible Wi-Fi and Bluetooth devices. People having electronic devices such as most cell phones, iPods, and computers carry unique information in their devices’ Wi-Fi and Bluetooth MAC address. This information can be encrypted and used anonymously to detect the origin and destination of pedestrians by matching data collected at entrances and exits from the system. Assuming that visible Wi-Fi and Bluetooth devices are uniformly distributed among the users of the system, it is possible to estimate a transit O-D matrix for the entire system not only on a daily basis but also over a time period allowing the agency analyze time-dependent OD demand for different location pairs. Moreover the same sensors proposed by the research teams will capture waiting times of the same sample of the users at fixed locations in each sensor location.

1.      PROJECT AND DATA DOCUMENTATION

It is possible to detect the proximity of Personal Electronic Device users utilizing Wi-Fi probe requests when their devices are looking for networks to join. Nowadays, everyone carries either a mobile phone or a smart device that is basically a handheld minicomputer with multiple means of data transmission such as Bluetooth and Wi-Fi.  Wi-Fi is enabled by default and whenever a user walks out of range of a connected Wi-Fi network, the device starts to make periodical probe requests which contains the unique MAC address of the client and scan available networks nearby (SSID).  In addition, these probe requests also contain the information of the networks that have been previously connected (ESSID). These requests make it possible to gather up this data and use it to track people.

Here is the full list of information fields that can be in a Probe Request (source IEEE 802.11-2012). Note that VHT capability element added to this list in 802.11-2013 (802.11ac) amendment.

 

There are mainly 2 modes in which Wi-Fi devices can operate.

·         Infrastructure mode

o   Mobile units communicate through an access point that serves as a bridge to other networks

·         Ad hoc mode

o   Mobile units transmit directly peer to peer

In the infrastructure mode, the device can be set to Monitor mode in which the device doesn’t advertise its presence while behaves like an access point to pick up all nearby traffic.

Hardware

Raspberry Pi: is a low cost small sized computer that is compatible with monitors or TVs. It uses a standard keyboard and mouse and supports object oriented programming languages such as Scratch and Python. It is capable of accomplishing tasks that one can expect a desktop computer to do. In addition, it is possible to build the Pi so that it will have the ability to interact with the environment. It has been used to measure the air quality, tweet birdhouses with infra-red cameras, as a weather, and a radio station.

PIR sensors: "Passive Infrared" or "IR motion" sensors sense motion. Everything in the world emits a small amount of infrared radiation. When something is hotter, it means more radiation is emitted. PIR sensors can detect a change in IR levels of their detection and therefore sense motion.
